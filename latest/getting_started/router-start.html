安装Router

Openshift的Router服务主要为Service提供外部访问的入口点，主要通过HAProxy(默认)和DNS服务完成代理和访问。

由于router的启动方式使用的是docker --host方式启动，使用宿主机的Net Namespace，所以我们需要相应的用户权限来启动router。

创建服务账号

该步骤不是必须的，在install router的时候应该也会自动创建。如果需要创建可以执行命令：

$ co create serviceaccount router -n default
在default的project中创建router服务账户， 平台的公共应用建议安装到default的project下。

为用户添加cluster-admin权限

首先我们需要对当前使用过的账户进行提权，给当前用户添加cluster-admin权限，执行命令：

$ oadm policy add-cluster-role-to-user cluster-admin admin --config=安装路径/openshift.local.config/master/admin.kubeconfig (1)
1 --config 参数需要指定master的admin.kubeconfig文件。

添加当前Project的router账户到hostnetwork

由于在执行创建的时候可能会提示需要执行此命令，如果提示则直接执行：

$ oadm policy add-scc-to-user hostnetwork -z router

添加服务账户到hostnetwork

将服务账户system:serviceaccount:default:router添加到hostnetwork的组中，执行命令：

$ oadm policy add-scc-to-user hostnetwork system:serviceaccount:default:router

创建router

准备工作做好后接下来就可以创建router了，执行命令：

$ oadm router myrouter --replicas=1 --service-account=router (1)
1 myrouter为router的名称，可以自定义。

执行后将会创建Service、DeploymentConfig、Pod并启动。

常见问题

  1. 创建router时，提示“Error from server: User "admin" cannot get securitycontextconstraints at the cluster scope”

    该提示主要是说当前的用户admin没有权限，需要进行提权操作，具体可以参照为用户添加cluster-admin权限。

  2. 创建router时，提示“Error from server: User "admin" cannot get clusterpolicybindings at the cluster scope”

    一般是在为用户添加cluster-admin权限时遇到，其中的--config参数是不能少的。

  3. 创建router时，提示“error: router could not be created; service account "router" is not allowed to access the host network on nodes, grant access with oadm policy add-scc-to-user hostnetwork -z router”

    信息后面已经给出操作提示，具体可以参照添加当前Project的router账户到hostnetwork。
