通过route访问应用

  • 创建Route
  • 配置使用Route
    • 使用hosts文件进行域名解析
  • 访问Route

我们可以对Service创建route配置，通过生成的url可以进行外网访问。 如果需要外网访问，需要我们搭建外网DNS服务，将域名的解析由我们搭建的DNS服务来进行解析，推荐使用Dnsmasq、mdns。

创建Route

首先我们可以在浏览器 控制台左侧菜单 ▸ Application ▸ Services 来查看Services的列表信息，点击将要创建 route 的 Service 查看详情信息。

图 1. demo Service详情

点击 Create route 跳转到route的创建界面。

图 2. route创建界面

我们保持默认选项就行，点击 Create 完成创建。

图 3. route创建结果

我们可以在浏览器 控制台左侧菜单 ▸ Application ▸ Routes 来查看routes的列表信息。

图 4. routes列表

配置使用Route

当route创建成功后，我们就可以在任何可直接访问 master 主机的网络中使用域名进行应用的访问了，如果使用的常用域名，比如 .com、.cn 等，需要配置DNS服务器为我们的DNS解析服务地址。 如果是自定义私有域名，那么则需要配置访问主机的DNS解析地址。

使用hosts文件进行域名解析

我们可以简单的通过访问主机的 hosts 文件来完成指定域名的解析，来进行一些测试，编辑系统的 hosts 文件。

Windows

C:\Windows\System32\drivers\etc\hosts

Linux

/etc/hosts

添加解析记录
192.168.80.132 docker-demo-mytest.router.default.svc.cluster.local  (1) (2)
1 IP地址为 master 的主机地址。
2 域名为我们创建的route地址。

添加完后，保存文件。
