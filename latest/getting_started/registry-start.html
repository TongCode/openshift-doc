安装Registry

由于我们使用openshift会进行代码的持续部署，而部署的结果最终会生成一个的docker image来进行执行，所以我们需要使用 openshift/origin-docker-registry 来搭建一个私有的Docker Registry来进行使用。 执行命令：

$ oadm registry --config=安装路径/openshift.local.config/master/admin.kubeconfig --service-account=registry (1)
1 --config 参数需要指定master的admin.kubeconfig文件

执行后将会自动创建 serviceaccount、添加role、创建DeploymentConfig、创建Service，并完成Register pod的启动。

配置Registry

安装并启动我们的Registry后，还需要让Docker来使用我们的私有Registry，下面主要针对Ubuntu和Centos来做下介绍。

Ubuntu系统的配置

我们需要编辑Docker的配置文件，默认在：/etc/default/docker

/etc/default/docker
DOCKER_OPTS=' --insecure-registry 172.30.0.0/16 ' (1)
1 --insecure-registry 参数用来指定私有仓库的信息。

配置完成后，我们需要重启docker服务，执行命令：

$ service docker restart

Centos系统配置

我们需要编辑Docker的配置文件，默认在：/etc/sysconfig/docker

/etc/sysconfig/docker
OPTIONS=' --insecure-registry 172.30.0.0/16 ' (1)
1 --insecure-registry 参数用来指定私有仓库的信息。

配置完成后，我们需要重启docker服务，执行命令：

$ systemctl daemon-reload
$ systemctl restart docker.service

验证配置

当我们修改完配置并重启服务后，需要来验证下我们的配置是否生效，执行命令:

$ docker info
图 2. docker信息

可以看到，我们的配置已经生效。
