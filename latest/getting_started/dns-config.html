DNS介绍

这里的DNS主要是指openshift使用skydns提供的解析服务，如果我们需要在集群内部通过域名来访问Service、pod、endpoints，则需要对相关主机进行nameservice的设置。

在openshift平台内部，各组件之间互相访问的默认规则为：

Table 1. Dns Example Names
Object Type Example

Default

<pod_namespace>.cluster.local

Services

<service>.<pod_namespace>.svc.cluster.local

Endpoints

<name>.<namespace>.endpoints.cluster.local
由于我们在openshift应用中，会通过组件的域名来进行访问，这样可以屏蔽掉ip地址的多变性。而域名的解析服务将会有skydns来完成，需要将所有使用组件域名访问的主机，dns的解析服务要用skydns。

配置Dns

编辑 /etc/resolv.conf 文件，这里需要配置两个 nameservice，其中一个为 master的IP ，另一个为公共可用的DNS服务地址。openshift默认会使用master的DNS服务进行解析，如果master的DNS无法匹配到，则会使用公共的DNS服务进行查询。

图 1. 配置结果示例
我们通过/etc/resolv.conf配置nameservice后，在系统重启后配置会被还原，这个情况是需要去修改/etc/resolvconf/resolv.conf.d/base文件。

测试Dns

配置完成后，我们需要测试下当前系统的dns解析服务是否是配置的结果，执行命令：

$ dig

可以看到回显信息中的 SERVER: 192.168.80.132#53(192.168.80.132) ，表示已经使用了我们的DNS服务进行查询了。
