创建Application

  • 通过Docker Image创建Redis服务
  • 通过Dockerfile创建demo应用
  • 参考文档

openshift支持项目创建的方式由很多种，我们可以根据情况选择适用的方式来完成创建。创建Application的方式主要有：

  • 通过Docker Image进行创建

  • 通过Dockerfile进行创建

  • 通过项目源码进行创建

  • 通过 Template 进行创建

后面我们将会对每一种方式进行介绍和使用，由于是快速应用，所以我们仅使用 通过Docker Image 和 通过Dockerfile 来完成Application的创建。

下面我们将通过两种方式分别创建redis和demo两个Application，redis Application会提供一个redis服务，而demo Application将会提供一个基于nodejs的Web项目，来访问redis服务。

通过Docker Image创建Redis服务

首先需要我们使用 Client 工具登陆到平台，然后执行命令：

$ oc new-app --docker-image redis (1) (2)
1 --docker-image 表示基于Docker Image来创建。
2 redis Docker Image的名称，如果是其他第三方的Image，需要指定完整的名称。
通过指定Docker Image的名称，会自动从当前Docker配置的仓库中去查询该Image是否存在，如果存在且本地没有下载过，则会自动进行docker pull操作，完成下载。

创建完成后，我们可以回到浏览器，通过UI控制台来查看redis的启动情况。

通过Dockerfile创建demo应用

通过Dockerfile来创建Application，其实也是通过项目源码或发布包创建，只是需要在项目中包含一个Dockerfile文件，其中描述了如何对当前的项目源码后者发布包进行部署运行。

我们已经准备好了一个测试项目，该项目主要发布在github上，访问地址为： https://github.com/TongCode/docker-demo ，项目使用Nodejs提供了一个web服务，然后进行访问次数的计数输出。

接下来我们在项目中创建Dockerfile文件

Dockerfile
FROM node (1)
RUN mkdir -p /var/www/docker-demo (2)
WORKDIR /var/www/docker-demo
COPY package.json package.json
RUN npm install --production (3)
COPY . .
EXPOSE 3000 (4)
CMD ["/bin/bash", "-c", "npm start"] (5)
1 FROM 标明我们将使用Docker官方提供的node Image来进行构建。
2 我们将会通过一些指令来完成项目的部署工作，比如创建目录、复制文件等。
3 熟悉nodejs的话，应该一下就明白了，这是在通过npm来下载依赖包。
4 通过EXPOSE来完成端口导出，表示我们可以通过3000端口来访问服务。
5 通过CMD来执行npm start启动。

项目准备好后，我们将使用项目的git地址来完成Application的创建，执行命令：

oc new-app https://github.com/TongCode/docker-demo.git (1)
1 https://github.com/TongCode/docker-demo.git 是我们项目的git地址，我们在new-app时可以直接指定项目的仓库地址。
执行创建后，会自动读取我们项目仓库中的文件，判断是否存在Dockerfile，然后会执行docker build来完成构建，生成新的Docker Image，将会自动通过docker push提交到我们的私有仓库中。

创建完成后我们可以通过查看Docker Image列表来看到构建后的Image，执行命令：

$ docker images

可以看到生成的image已经提交到仓库中，如图中的172.30.218.206:5000/mytest/docker-demo所示。
